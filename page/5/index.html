各种架构

感知运动模型的七种变体，展示了骨干架构（Transformer，VLM）与动作头（离散令牌、扩散、流匹配）的不同组合。

  1. Transformer + discrete action

    • 在 VLA 模型的早期阶段（如 RT-1 或 Gato），将 Transformer 用于机器人控制的核心思路是：把机器人的动作（Action）看作是一种“语言”。

    • 自回归生成 (Autoregressive): 模型不是一次性预测出完整的 7 维动作，而是按顺序预测：先预测 \(x\)，再预测 \(y\)，再预测 \(z\)...。这样每一维度的预测都可以参考前面的信息。

    • ```python import torch import torch.nn as nn

      class SimpleVLA(nn.Module): def init(self, vocab_size=256, action_dim=7): # 7维动作: 6自由度+抓取 super().__init__() # 1. 视觉编码器 (例如 EfficientNet 或 ViT) self.vision_encoder = nn.Sequential(nn.Conv2d(3, 64, 3), nn.AdaptiveAvgPool2d((1, 1)))

          # 2. 文本编码器 (例如冻结的 CLIP 或 BERT)
    self.text_encoder = nn.Linear(512, 128) 

    # 3. 动作嵌入 (将离散的 Action ID 转为向量)
    self.action_embedding = nn.Embedding(vocab_size, 128)

    # 4. 标准 Transformer Decoder
    self.transformer = nn.TransformerDecoder(
        nn.TransformerDecoderLayer(d_model=128, nhead=8), num_layers=6
    )

    # 5. 输出层: 预测每个动作维度的分类结果 (256个分桶)
    self.action_head = nn.Linear(128, vocab_size)

def forward(self, image, text_features, past_actions):
    # image: [Batch, 3, 224, 224]
    # text_features: [Batch, 512] (预先提取的语言特征)
    # past_actions: [Batch, Sequence_Length, action_dim] (之前的动作 ID)

    # 提取多模态特征
    vis_feat = self.vision_encoder(image).flatten(1) # [B, 128]
    lang_feat = self.text_encoder(text_features)     # [B, 128]

    # 组合成 Token 序列
    # 假设我们将 [语言, 视觉, 历史动作] 拼在一起作为输入
    obs_token = (vis_feat + lang_feat).unsqueeze(1)  # [B, 1, 128]
    act_tokens = self.action_embedding(past_actions) # [B, T, 128]

    # Transformer 处理
    output = self.transformer(act_tokens.transpose(0, 1), obs_token.transpose(0, 1))

    # 预测下一个动作的概率分布
    logits = self.action_head(output.transpose(0, 1)) # [B, T, 256]
    return logits

  2. Transformer + diffusion action head（diffusion action head结构可以是mlp可以是unet可以是dit）

  • 不对action空间切分，还是连续的。

    transformer不直接输出动作，而是输出一个环境表征condition,然后送入一个diffusion model进行多部去噪

  • 将机器人控制变成一个条件生成（Conditional Generation）任务，直接建模多峰分布（Multimodal Distribution，即同一个场景下，机械臂既可以从左边绕过去拿杯子，也可以从右边拿）。

  • Diffusion Policy (TRI团队), π0 (Physical Intelligence), 各种基于 DiT 的具身模型。

  • 动作平滑，解决多峰问题，但是延迟高

  • import torch
import torch.nn as nn

class Simple1DDiffusionNet(nn.Module):
    """一个简化的 1D 扩散去噪网络，用于预测噪声"""
    def __init__(self, action_dim, cond_dim):
        super().__init__()
        # 输入：当前带噪动作 + 时间步嵌入 + Transformer传过来的条件嵌入
        self.net = nn.Sequential(
            nn.Linear(action_dim + 1 + cond_dim, 256),
            nn.ReLU(),
            nn.Linear(256, 256),
            nn.ReLU(),
            nn.Linear(256, action_dim) # 预测的噪声形状与动作一致
        )
    def forward(self, noisy_action, timestep, cond):
        # 拼接所有输入
        inputs = torch.cat([noisy_action, timestep, cond], dim=-1)
        return self.net(inputs)

class TransformerDiffusionActionPolicy(nn.Module):
    def __init__(self, d_model=512, action_dim=7, pred_horizon=8):
        super().__init__()
        # 1. 骨干网络：提取环境和历史特征
        encoder_layer = nn.TransformerEncoderLayer(d_model=d_model, nhead=8, batch_first=True)
        self.transformer = nn.TransformerEncoder(encoder_layer, num_layers=4)

        # 2. 扩散去噪网络
        self.pred_horizon = pred_horizon # 扩散模型一次性预测未来的动作步数 (Action Chunk)
        self.action_dim = action_dim
        self.diffusion_head = Simple1DDiffusionNet(action_dim=action_dim * pred_horizon, cond_dim=d_model)

    def forward(self, obs_tokens, noisy_action, t):
        """
        训练阶段的 forward：
        noisy_action: 真实动作加噪后的状态 (batch, pred_horizon * action_dim)
        t: 当前扩散步数，如 0~50 之间的整数 (batch, 1)
        """
        # 1. Transformer 提取特征作为 Condition
        cond_features = self.transformer(obs_tokens)[:, -1, :] # (batch, d_model)

        # 2. 扩散头预测噪声
        pred_noise = self.diffusion_head(noisy_action, t, cond_features)
        return pred_noise # 用于与真实噪声计算 MSE Loss

    @torch.no_grad()
    def predict_action(self, obs_tokens, num_diffusion_steps=10):
        """推理阶段：从纯高斯噪声开始，经过多步迭代去噪，生成连续的轨迹"""
        batch_size = obs_tokens.size(0)

        # 1. 提取环境特征（在去噪循环外只执行一次，节省算力）
        cond_features = self.transformer(obs_tokens)[:, -1, :] # (batch, d_model)

        # 2. 随机初始化一串纯高斯噪声作为动作起点
        current_action = torch.randn(batch_size, self.pred_horizon * self.action_dim).to(obs_tokens.device)

        # 3. 循环迭代去噪 (DDIM 或 DDPM 采样逻辑简化版)
        for step in reversed(range(num_diffusion_steps)):
            # 构造时间步输入
            t_tensor = torch.full((batch_size, 1), step, dtype=torch.float32).to(obs_tokens.device)

            # 预测噪声
            pred_noise = self.diffusion_head(current_action, t_tensor, cond_features)

            # 根据扩散理论更新当前动作（此处为极简线性步进示意）
            current_action = current_action - 0.1 * pred_noise 

        # 4. 将扁平的向量恢复为 (batch, 预测步数, 动作维度) 的连续轨迹
        action_trajectory = current_action.view(batch_size, self.pred_horizon, self.action_dim)
        return action_trajectory

3.diffusion transformer

  • 噪声动作直接被转化为 Token，和其他多模态 Token（视觉、语言）一起排队进入 Transformer。
  • 它不仅要理解环境，还要直接负责去噪。每一层 Transformer 都在处理“环境特征如何引导噪声还原为动作”。整个 Transformer 骨干网络本身就是去噪器。它利用 Self-Attention 来建模“当前视觉特征”与“正在生成的动作轨迹”之间的复杂空间关系。

[!NOTE]

将大量的网络数据整合到机器人端到端控制中让机器人操作能涌现出泛化能力。

methods:

  • 预训练的vlm + finetune:互联网规模数据(图像-文本对)+机器人演示数据

    • 为什么有了vlm还要用视觉问答损失？防止忘记通用知识

  • 直接把机器人动作映射为language的同等地位的token，因此架构基本还是vlm的结构。输入是 [256个视觉Token] + [提示词Token]，输出是 [7-8个动作Token]（代表机械臂的 \(x,y,z,\text{roll},\text{pitch},\text{yaw},\text{gripper}\) 离散化 Bin 值）。因为输出序列极短，KV Cache 的压力非常小，核心瓶颈在 Prefill（首字延迟，即视觉 ViT 的前向传播速度）。

  • 推理频率：

    • 55B 旗舰版：在谷歌云端多 TPU 协同下，控制频率仅能达到 1 - 3 Hz（每秒控制 1-3 次），对于高精度、高速率装配来说较为勉强。

      5B 剪裁版：控制频率可提升至 5 Hz 左右，基本能满足日常移动抓取任务。

  • 训练 RT-2 采用的是 Co-fine-tuning（共同微调） 策略：70% 的算力喂机器人操作数据（如 RT-1 遗留数据集），30% 的算力继续喂互联网 VQA（视觉问答）数据，防止大模型失忆。

pi07: 具有涌现能力的可控模型

URL:

https://www.pi.website/blog/pi07

量子位总结：https://mp.weixin.qq.com/s/Z7JpjZbEk59wqAKCLLbKtg

架构：

π0.7把prompt展开成四层：任务指令（清理厨房）+子任务指令（打开冰箱）+子目标图像（下一秒画面应该长什么样）+episode元数据（这条数据质量几分、有没有出错、速度多快）。

几个重点发现

  • 开箱即用no finetune:π0.7没做任何专项训练，就能在做咖啡、叠衣服、装箱三个复杂任务上，追平π0.6经过微调的的专家模型。

    • vla开始展现出组合泛化的能力，不一定强于world model？

  • Cross-embodiment：让UR5e双臂完成没有采集过的叠衣服任务

  • 通过metadata告诉“数据好坏”可以提升模型效果

    • 这点和机器学习很像，在判别模型里我们的“坏数据”可能有比好数据更高的价值

  • pi07的世界模型只预测把任务指令翻译成成功那一帧应该长啥样。不预测动作后果，不模拟物理，不参与决策链路。

技术实现：vlm + action expert + flow matching

π0.7是一个5B参数的模型，分三块。

  • VLM骨干：4B参数的Gemma3，负责理解视觉和语言。
  • Action expert：860M参数的transformer，用flow matching生成连续动作chunk，50Hz高频控制。
  • World model：从14B的BAGEL图像生成模型初始化，负责给π0.7画出未来几秒应该是什么样子。

在推理中，模型输入包括：4路摄像头（前视+两个腕部+可选后视）、每路6帧历史画面、机器人关节状态、再加上任务指令、子任务指令、元数据、以及world model实时画出的次目标图像。

输出是一段50步的action chunk，实际执行15到25步，然后再推下一段。

现方向根据下流任务各异

在 Next-Best-View (NBV) 和主动视觉领域，很难定义一个单一的“绝对 SOTA”，因为性能高度依赖于任务目标（如纯几何重建、物体 6D 位姿估计或抓取操作）。

sota

根据 2024-2025 年的最前沿进展，以下是各细分方向公认的顶尖（SOTA）算法或框架：

1. 纯物体 3D 重建与视角规划：DM-OSVP++

Nerf 2025年4月 德国波恩大学人形机器人实验室

方法：one-shot view planning 代码暂时未开源：https://github.com/psc0628/DM-OSVPplusplus demo: https://www.youtube.com/watch?v=GBBCn28v-lQ

2.实时感知决策：## GauSS-MI

3D Gaussian Splatting 与 香农互信息 (Shannon Mutual Information) 结合。 2025年7月 开源：https://github.com/JohannaXie/GauSS-MI

demo:https://www.bilibili.com/video/BV1rVNUziEFy/?vd_source=db693e375cf7809ea90067d2fd609b51

CUDA模块代码：https://github.com/JohannaXie/diff-gaussian-rasterization-gaussmi

3.针对无纹理物体的 6D 位姿估计

Active 6D Pose 多伦多大学航空航天研究所和机器人研究所 2025年3月 未开源

在工业场景或处理反光、无纹理物体（这是位姿估计最难的部分） 通过 多视角 RGB 融合 来消除单视角下的对称性歧义。它通过主动移动相机来寻找能最大程度降低位姿估计熵（Entropy）的视角。 机械臂在杂乱堆叠（Clutter）中抓取金属零件或塑料制品。

4.任务为导向：AC-NBV

23年11月

  • 核心优势：它是示能引导（Affordance-driven）的。它不追求物体表面的 100% 重建，而是优先寻找对“成功抓取”最有帮助的视角（例如物体的边缘、手柄、开口处）。
  • 结果：在抓取成功率（Success Rate）和效率上优于纯几何重建的 NBV。

和project manager, 产品经理如何沟通

Posted on 2026-04-14 Edited on 2026-06-11 In 个人日志

​ 在技术研发里，会有很多个角色，project manager项目经理，向程序员提需求的产品经理。我观察自己的心态发现，有时候我会很不爽，从而有一定的内耗。这种不爽的想法来自于：

  • pm等呈现一种push的角色，占据我的时间表，嘴一张就是一个ddl。
    • 因为我的目标之一是成为一个能把项目做成做好的合作者，所以接受pm的这种工作模态，忍受或者爆发都不是我想要的答案。
    • 我在思考如果我是pm的角色，如何和算法技术沟通，会让对方更愿意配合我
  • 除了催进度，pm无法提供实质性的帮助，却可以"lead"整个项目的需求。
  • 催出结果引发我的焦虑。我总有种“对结果负责的”焦虑感，但是技术研发总会有各种各样的问题，害怕达不到expectations也会让我焦虑。因此我会把这种焦虑投射到pm身上，以反感或者厌恶的方式“害怕pm提出目标”（仔细想来应该是有优绩主义的影响）
    • 这部分还是要自己去解决。涉及到很多的议题：
      • 如何管理自己的期望
      • 如何管理别人的期望
      • 做事情的内生动力和内燃机应该是什么样的

​ 下面从两个role去阐述：

  1. 如果我是算法研发ae——核心“反向利用pm”
    1. 关于觉得pm只能“催进度”，认识到现在大多数企业的pm都是不合格的，我的不爽也是正常的（并且是健康的）。可以参考角色转换——如果我是一个pm为什么我会做的更好
    2. 变“被动接受”为主动管理，在项目初期就要给出概率评估。
    3. 利用pm：优秀的ae要把pm当成自己的“外挂”。当你缺算力、缺标注数据、或者跨部门对接推不动时，PM 是帮你解决这些琐事的最佳人选。
  2. 如果我是项目经理：从“supervise”变成赋能
    1. 要有向下兼容，对算法有了解。共担风险，而非转嫁压力.
    2. 明确项目的中间态和中间成果，让“对齐”是必要的。
    3. 建立“容错空间”，考虑技术研发的曲折性。尊重技术的“颗粒度”.
    4. 要向技术解释背后的商业逻辑、用户需求。AE 认为“我在做一个牛逼的产品”而不是“我在完成一个 PM 的任务”时，内驱动力会大幅提升。

算法工程师的“不爽”，通常是因为在项目链条中感到自己被工具化了。

对抗这种心态最有效的武器，是参与决策。 当你开始主动去评估项目可行性、去反向要求资源、去界定技术边界时，你就不再是一个“听活儿的”，而是一个带头人。

介绍

机器人动力学关注的是作用在机器人上的力与其所产生的加速度之间的关系。

两种：

  • Fixed-base

  • Floating-base

    Kinematics

    运动学只关心“怎样运动”（位置、速度、加速度、旋转角度等几何和时间上的描述），而不关心“为什么运动”（即不关心产生运动的力、质量、扭矩等）。

    position

    笛卡尔坐标系、柱坐标系、球坐标系

velocity:

正运动学

  • 正运动学解决的是“如果我知道每个关节转了多少，那么机械臂末端在哪里？”的问题。

  • \[ \chi_e = \chi_e(q) \\ q 是关节向量，\chi_e 是末端执行器在笛卡尔空间（或操作空间）中的位姿。 \]

微分运动学Differential Kinematics

  • 微分运动学研究的是速度之间的映射，即“如果关节以某种速度运动，末端执行器的线速度和角速度是多少？”。

  • \[ \dot{\chi}_e = J_{eA}(q) \dot{q} \\ \dot{\dot{q}} 是关节速度向量，\dot{\chi}_e 是末端速度向量。 \]

  • 线性映射：与正运动学的非线性不同，在给定的位形 \(q\) 下，末端速度与关节速度之间是线性映射关系，这个映射矩阵就是 雅可比矩阵 (Jacobian Matrix)。

  • 雅可比矩阵 \(J_{eA}(q)\) 的维度是 \(m \times n\)（\(m\) 为任务空间维数，\(n\) 为关节数）。其每一个元素代表了第 \(j\) 个关节的变化对末端第 \(i\) 个坐标分量的偏导数： \[ J_{eA}(q) = \frac{\partial \chi_e}{\partial q} \]

  • 通常将雅可比矩阵拆分为线性部分（位置）*和*旋转部分（姿态）： \[ J_{eA} = \begin{bmatrix} J_{eA}^P \\ J_{eA}^R \end{bmatrix} \\ J_{eA}^P$ (Position Jacobian) :对应末端执行器的线速度 v_e = J_{eA}^P \dot{q}\\ J_{eA}^R$ (Orientation Jacobian)：对应末端执行器的角速度 \omega_e = J_{eA}^R \dot{q} \]

几何雅可比/基础雅可比

  • 几何雅可比矩阵 \(J_{e0}(q)\) 直接将关节速度 \(\dot{q}\) 映射到末端执行器在三维空间中的物理速度 \(w_e\)（即线速度 \(v_e\) 和角速度 \(\omega_e\)）

  • 几何雅可比和分析雅可比的关系

    • 线速度部分： 通常情况下，解析雅可比的位置部分 \(J_{eAP}\) 和几何雅可比的线速度部分 \(v_e\) 是一致的，即 \(v_e = \dot{\chi}_{eP}\)。
    • 旋转部分（差异所在）： * 解析雅可比使用的是姿态表示（如欧拉角）的时间变化率 \(\dot{\alpha}\)。
      • 几何雅可比使用的是物理角速度 \(\omega_e\)。

逆微分运动学

在机器人运行过程中，我们通常给末端设定一个期望速度 \(\dot{w}_e\)。

  • 正向微分运动学： \(\dot{w}_e = J \dot{q}\)（已知关节动多快，算末端多快）。
  • 逆微分运动学： \(\dot{q} = J^{+} \dot{w}_e\)（已知末端要多快，反推关节给多少速度）。

这里使用的是伪逆（Pseudo-inverse） \(J^{+}\)，因为雅可比矩阵通常不是方阵。

对偶性是优化理论里非常深刻的一个概念。

原问题关注如何在约束范围内找到最小的目标值

对偶问题关注如何通过寻找目标值的“下界”来逼近最优解

  1. 标准形式： \[ \min\limits_{x} f_{0}(x) \\ s.t. f_{i}(x) \le 0, i=1,...,m \\ h_{i}(x)=0, i=1,...,p \]

    1. 形式： \[ L(x,\lambda, \nu) = f(x) + \sum_{i=1}^{m}\lambda_{i}g_{i}(x)+\sum_{i=1}^{p}\lambda_{i}h_{i}(x) \]

      1. 前面的系数叫朗格朗日乘子
      2. 你可以把“价格”或者“敏感度”，如果某个约束难以满足的时候.

    2. 对偶函数定义为： \[ g(\lambda, \nu)=\inf_{x}L(x,\lambda,\nu) \\ 对偶函数是朗格朗日函数关于x的下确界 \\ 无论原问题是否凸，对偶函数永远是凹函数。意味着更好求解。\\ 下界属性：对于任何\lambda>0,对偶函数的值永远不会超过原问题的最优值p^*，即g(\lambda,\mu)\le p^* \] 对偶问题就是尝试找到哪个最好的下界： \[ \max g(\lambda,\nu) \\s.t. \lambda >= 0 \]

KKT条件: 最优解的充分判定标准

如果一个问题是凸优化问题，只要满足kkt条件，这个点就一定是全剧最优解。 \[ 1.定常性：\nabla f_0(x^*) + \sum_{i=1}^{m} \lambda_i^* \nabla f_i(x^*) + \sum_{j=1}^{p} \nu_j^* \nabla h_j(x^*) = 0 \\ 2.原问题可行性：f_i(x^*) \le 0, \quad i=1, \dots, m \\ h_j(x^*) = 0, \quad j=1, \dots, p \\ 3.对偶可行性：不等式约束的朗格朗日乘子非负：\\ \lambda_i^* \ge 0, \quad i=1, \dots, m \\ 4.互补松弛性：\lambda_i^* f_i(x^*) = 0, \quad i=1, \dots, m\\ 最精华的一条：如果f_{i}(x)<0,\lambda_{i}=0，已经满足了，没有贡献；如果\lambda_{i}>0, \\这个约束在起作用，那么f_{i}(x)=0，最优解就在边界上。 \]

数值线性代数

将抽象的线性代数理论转化为计算机可执行算法的学科。在机器人算法（如位姿估计、动力学模拟）中，我们几乎从不直接求矩阵的逆 \(A^{-1}\)，而是通过矩阵分解（Matrix Factorization） 来高效、稳定地求解方程。

  1. LU分解

    1. \(A = LU\)
    2. 将矩阵拆分为一个下三角矩阵 \(L\)（对角线下方有值）和一个上三角矩阵 \(U\)（对角线上方有值）。
    3. 需要解多次 \(Ax = b\)（\(A\) 不变，\(b\) 在变），先做一次 LU 分解会大幅节省计算量。

  2. QR 分解

    1. \(A = QR\)
    2. \(Q\) 是正交矩阵（旋转/镜像），\(R\) 是上三角矩阵。
    3. 是处理最小二乘法的最稳健工具。
    4. 在你的 SfM（运动恢复结构）或点云配准中，当观测数据带噪声且方程组超定时，QR 分解比直接用正规方程更不容易产生数值误差。

  3. Cholesky 分解

    1. \(A = LL^T\)
    2. \(A\) 必须是对称正定矩阵。
    3. 在 卡尔曼滤波 (EKF/UKF) 或 二次规划 (QP) 中，协方差矩阵和 Hessian 矩阵通常是对称正定的，Cholesky 是最优的选择。

  4. 奇异值分解 (SVD - Singular Value Decomposition)

    1. \(A = U\Sigma V^T\)

    2. 将矩阵看作一种线性变换，拆解为：旋转 (\(V^T\)) \(\rightarrow\) 拉伸 (\(\Sigma\)) \(\rightarrow\) 旋转 (\(U\))。

    3. 是线性代数中的“瑞士军刀”。\(\Sigma\) 对角线上的奇异值告诉你矩阵在各个方向上的强度。

    4. 求解本质矩阵 \(E\) 和基础矩阵 \(F\)：你之前提到的 8 点算法，核心步骤就是对矩阵进行 SVD 并将最小奇异值设为 0（强制 Rank-2 约束）。

      主成分分析 (PCA)：提取机器人视觉特征的主要特征。

      求伪逆：处理病态矩阵（Condition number 极差）时最可靠的方法。

  2. 牛顿法

    如果说梯度下降是“只看脚下”，那么牛顿法就是“利用地图建模”。

    核心思想：在当前点 \(x\) 处，利用二阶泰勒展开对函数进行局部近似，将其看作一个二次函数（抛物面）。 \[ f(x + \Delta x) \approx f(x) + \nabla f(x)^T \Delta x + \frac{1}{2} \Delta x^T \nabla^2 f(x) \Delta x \]

​ 更新公式： \[ \]x^{(k+1)} = x^{(k)} - (^2 f(x{(k)})){-1} f(x^{(k)})\[ \]

​ 几何解释：牛顿法每一轮迭代实际上是跳到了这个局部二次近似函数的最低点。
