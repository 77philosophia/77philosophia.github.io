极大似然估计(MLE)

寻找一组参数，使得这组参数下观测到当前数据的概率最大。

如果数据是完整的即没有隐藏变量，所有特征和标签都已知），MLE 的求解通常是一个直接的优化问题。

核心流程：四步走策略

假设我们有一组独立同分布（i.i.d.）的数据 \[$X = \{x_1, x_2, \dots, x_n\}\]$，我们要估计模型参数 \(\theta\)。

第一步：构建似然函数 (Likelihood Function)

似然函数 \(L(\theta)\) 表示在参数 \(\theta\) 下，观测到整组数据 \(X\) 的联合概率。由于数据是独立的，联合概率等于各样本概率的乘积：

\[L(\theta) = P(x_1, x_2, \dots, x_n | \theta) = \prod_{i=1}^{n} P(x_i | \theta)\]

第二步：取对数似然 (Log-Likelihood)

由于连乘运算在求导时非常麻烦，且容易导致数值下溢（很多概率相乘会变得极小），我们通常对 \(L(\theta)\) 取自然对数。由于 \(\ln(x)\) 是单调递增函数，使 \(\ln L(\theta)\) 最大的 \(\theta\) 同样会使 \(L(\theta)\) 最大：

\[\ell(\theta) = \ln L(\theta) = \sum_{i=1}^{n} \ln P(x_i | \theta)\]

这样，乘法变加法，大大简化了计算。

第三步：求导并令其为零

为了找到极值点，我们对参数 \(\theta\) 求偏导（即计算梯度），并令导数为 0：

\[\frac{\partial \ell(\theta)}{\partial \theta} = 0\]

第四步：求解参数

解上述方程，得到的解 \(\hat{\theta}_{MLE}\) 即为极大似然估计值。

2. 经典案例：高斯分布的参数估计

在机器人动作建模中，我们经常假设动作噪声服从高斯分布 \(N(\mu, \sigma^2)\)。如果我们有 \(n\) 个位置观测值，如何求均值 \(\mu\)？

  1. 写出单个样本的概率密度：\(P(x_i|\mu, \sigma) = \frac{1}{\sqrt{2\pi}\sigma} e^{-\frac{(x_i-\mu)^2}{2\sigma^2}}\)

  2. 写出对数似然：

    \[\ell(\mu, \sigma) = \sum_{i=1}^n \left( -\ln(\sqrt{2\pi}\sigma) - \frac{(x_i-\mu)^2}{2\sigma^2} \right)\]

  3. 对 \(\mu\) 求导：

    \[\frac{\partial \ell}{\partial \mu} = \sum_{i=1}^n \frac{x_i - \mu}{\sigma^2} = 0\]

  4. 得出结果：

    \[\hat{\mu} = \frac{1}{n} \sum_{i=1}^n x_i\]

    这证明了在正态分布假设下，算术平均值就是最科学的极大似然估计。

强化学习算法概览

Posted on 2026-03-21 Edited on 2026-04-07 In 专业笔记

这是一个根据提问和关键词展开rl领域的outline文档。

介绍

  • 理解强化学习的两个关键概念：trial-and-error search 和 delayed reward

Markov Decision Process

  • 五元祖的含义（S, A, P,R, y）

  • Returns的公式： Gt= Rt+1 + Rt+2+...

  • (state) Value function: 在state=s的时候Gt的期望值 V(s) \[ V^\pi(s) \]

  • (State-action) value function: \[ Q^\pi(s,a) \]

  • Bellman Equation: 此时的value function = 下一时刻所有的immediate reward和下一state的value和。

[!IMPORTANT]

注意，\[V^\pi(s)\] 和\[Q^\pi(s,a)\] 不是固定的，是不断优化的，所以上标有pi.

如何解决一个MDP问题：

1.动态规划

环境状态转移概率和reward function都已知，用动态规划算法即可。每一步选择最大的reward。

2. 蒙特卡罗方法

不知道环境的状态(model-free);直接从experience学习，执行完一次实验后计算，no bootstrapping；因此是无偏的。

3.Temporal-Difference Learning
  • 结合了动态规划和蒙特卡罗，基于当前状态进行估计更新；
  • Model-free:不知道MDP转移概率和reward;
  • 可以从incomplete experience中学习：on-line, continual learning
  • Sarsa: On-policy TD control, 每一次更新用实际的Q值去更新；
  • Q-learning: off-policy TD control,每一次的Q值更新不用实际走的action，而是argmax去更新；因为不是用的真实的q，因此叫off-policy

[!IMPORTANT]

在model-free情况下，一般估计Q(s,a), V(s)一般用于评估。

4. function approximation

  • 如果state space too large,可以用线性函数去拟合得到value function.

  • Deep Q-Network(DQN)：

    • 一句话：用神经网络估计Q(s,a)

    • 直接让神经网络去估计，存在一个问题，mdp的数据不是i.i.d.的，每一步和上一步相关；因此DQN引入eperience replay:存储experience然后mini-batch采样；

    • 网络“左脚踩右脚”的更新不稳定。用Fixed Q-target: 两个Q网络解决网络震荡。

    • \[ update: L(\theta) = E_{s,a,r,s'~U(D)}[(r+\gamma \max_{a}Q(s’,a‘,\theta^-)-Q(s,a;\theta)^2)] \\ \theta^-是target network,固定一定步数之后再更新 \]

5. Policy gradient Methods
  • 直接学习policy，而不是先学value function再deriing the policy.
  • 一个核心思想：最大化reward,更新的gradient是：
  • Natural Policy Gradient:
    • 对策略的梯度乘以Fisher information matrix
6.Actor-Critic Methods
  • policy-based方法高方差，value-based方法高bias。actor-critic方法利用这两种方法来更新。
  • actor网络更新：
  • value func更新：
    • 网络估计的是v(s),而不是q(s,a)
    • 给policy更新的是A(s,a)=q-v
7. Deterministic Policy gradient

  • 建模 \[ a = \mu_{\theta}(s) \]

  • 然后用链式求导法则进行更新

8. Deep Deterministic Policy gradient

  • 引入深度神经网络学习Q-function和 \[ \mu_(s) \]

  • Experience replay

  • 更新公式

9. relative Entropy Policy Search
  • 基于policy-based的优化前提（maxJ()）,避免策略前后差异过大（KL散度超过一个值）
  • 有约束的优化问题（朗格朗日乘子解决）
10.PPO
  • 目的也是为了防止过大步子的更新
  • 计算策略前后的一阶比值
11. Soft Actor-Critic
  • Off-policy actor-critic方法
  • 在最大化回报的基础上+系数*policy的熵，要求熵也最大化。

关于强化学习的On-policy 和Off-policy
  • 核心矛盾是探索exploration和利用exploitation的矛盾
  • On-policy:在 On-policy 算法（如 SARSA, A2C, PPO）中，更新公式里的 \(a\)（当前动作）和 \(a'\)（下一时刻动作）全部来自于当前的 \(\pi\)。
    • 当你一边玩游戏一边改进策略时，你当前的每一个动作都在反映你当下的认知。
    • 更新非常平滑、稳定，因为它始终在优化“我此时此刻的行为”。
    • 一旦你的认知（策略）更新了一点点，你之前所有的游戏录像（经验）就作废了。因为那些旧动作不再代表你现在的水平，如果强行学习旧数据，你的认知会被带偏。
  • Off-policy:我学习的是“理想策略”，但我执行的是“带有随机性的探索策略”。
    • 在 Off-policy 算法（如 Q-Learning, DQN, SAC）中，我们通常有两个策略：
      1. 行为策略 (\(\mu\))：负责到处乱跑，采集数据，存入“经验回放池”。
      2. 目标策略 (\(\pi\))：负责从池子里的陈年旧账中总结规律，进化成最强形态。
    • 它是怎么来的： 为了解决 On-policy 浪费数据的问题，科学家引入了贝尔曼最优方程。它发现：环境的物理规律（Q值）是客观存在的。 比如“在悬崖边往左跳会摔死”，这件事不论你是新手还是高手，不论你是今天跳还是昨天跳，结果都是一样的。
    • 容易不稳定。因为你是在拿“别人的/过去的”经验来指导“现在的”自己，如果处理不好（比如没有 Target Network），容易产生数值爆炸。
    • 数据利用率极高，可以学习人类专家的录像，也可以学习自己半年前的黑历史。
  • 如果是在 Isaac Sim 这种仿真环境里，PPO (On-policy) 通常是首选，因为采样不要钱且稳；但如果要上实机驱动，大家往往会选 SAC (Off-policy)，因为实机跑一次的成本太高了。
