富爸爸穷爸爸

Posted on 2026-05-21 Edited on 2026-05-23 In Books

TL;DR

​ 作者罗伯特·清崎（Robert Kiyosaki）通过对比两个父亲的观念——“穷爸爸”（亲生父亲，高学历的政府官员，代表传统工薪思维）和“富爸爸”（好朋友的父亲，早早辍学却通过商业和投资发家，代表富人思维）——拆解了实现财务自由的核心逻辑。

key sights

1. 资产 vs 负债：最根本的财富定义

很多人把房子、汽车当作资产，但清崎给出了一个极度简化却直击本质的定义：

  • 资产（Asset）： 能把钱放进你口袋的东西（如：出租房、股票、分红、能够自动运转的公司）。
  • 负债（Liability）： 把钱从你口袋拿走的东西（如：自住房、汽车贷款、信用卡账单）。

富人买入资产，穷人只有支出，中产阶级则买入他们以为是资产的负债。

2. 穷人为钱工作，富人让钱为自己工作

  • “穷爸爸”思维（老鼠赛跑）： 努力读书 \(\rightarrow\) 找份好工作 \(\rightarrow\) 升职加薪 \(\rightarrow\) 消费升级（买大房子、好车） \(\rightarrow\) 债务增加 \(\rightarrow\) 必须更努力地工作。这是一种被“恐惧”（害怕没钱）和“欲望”（想享受）驱动的恶性循环。
  • “富爸爸”思维： 工作是为了学习技能（销售、管理、投资），而不是为了眼前的薪水。通过工作积累原始资金后，迅速投入到资产中，让资产产生的被动收入（如利息、租金）来支付日常开销。

3.真正的财务自由：财商的四大基石

清崎强调，想要跳出“老鼠赛跑”，必须培养自己的财商（Financial IQ），它由四个维度的知识构成：

  • 财务知识： 读懂财务报表的能力，分得清真正的资产和负债。
  • 投资策略： 让钱生钱的科学与技巧。
  • 市场规律： 了解供给与需求，能识别出当下的投资机会。
  • 法律法规： 尤其是利用公司（Corporation）这一法律工具来合理避税和保护资产（富人的做法是：赚钱 \(\rightarrow\) 消费 \(\rightarrow\) 缴税；而工薪族是：赚钱 \(\rightarrow\) 缴税 \(\rightarrow\) 消费）。

4. 克服走向财富的五大障碍

很多人懂了道理依然不富裕，是因为无法克服人性中的弱点：

  • 恐惧： 害怕亏钱。富人不是不害怕，而是懂得如何管理风险，把失败当成学费。
  • 愤世嫉俗： 总是怀疑和抱怨（比如“买房肯定会崩盘”），从而错失机会。
  • 懒惰： 表现为“用忙碌来逃避思考”，不肯花时间研究财务。
  • 不良习惯： 先支付别人（还账单），最后才支付自己（投资）。富人会先支付自己，逼迫自己去创造新的财富。
  • 傲慢： 傲慢就是“无知+自大”。不知道的事情就要去学习，而不是假装它不重要。

一些习惯：

1. 改变看世界的“句式”：从消极防御到主动求解

普通人遇到向往但昂贵的东西时，潜意识的第一反应是：“我买不起。”

💡 富爸爸的思维训练： 永远把这句话换成：“我怎样才能买得起？”

  • 为什么： “我买不起”会让你的大脑瞬间停止思考，进入认命状态；而“我怎样才能买得起”是一个疑问句，它会强迫你的大脑开始疯狂运转，去寻找解决方案（找副业、优化资产、提升技能、发现新商机）。

2. 重新审视你的账单：区分“消费”与“投资”

每天花钱的时候，在心里做一次快速的分类。问自己这个东西到底是在把你口袋里的钱拿走（负债/消费），还是未来能把钱放进你的口袋（资产/投资）。

我们可以把日常支出做一个简单的重新定义：

  • 纯消费/负债： 奶茶、最新款手机、面子消费。这些东西落地就贬值。
  • 自我投资（通往 B/S 象限）： 购买能提升个人生产力的硬核工具、前沿技术课程。这种支出能放大你未来的时薪或让你具备构建系统的能力。
  • 资本投资（通往 I 象限）： 哪怕每个月只剩几百块，也尝试把它投入到你真正研究过的优质公司股票、指数基金或能产生利息的资产中。投资的习惯远比投资的金额重要。

3. 锻炼“看懂资产”的眼光：日常的商业拆解

走在路上或者刷手机时，不要只做一个单纯的消费者，试着用 B 象限（企业主）的视角去拆解你看到的每一个生意：

  • 看到一家生意火爆的跨国连锁餐厅：不要只看菜好不好吃，去观察它的系统（SOP）——它是如何标准化的？员工是如何分工的？离开了创始人，这个店为什么还能开到全世界？
  • 看到一个成功的自媒体或独立工具：去拆解它的变现闭环——它的流量从哪里来？它解决了什么痛点？它的自动化交付流程是怎么做的？

训练目标： 让你具备“穿透表象看底层系统”的能力。只有看懂了别人的系统，未来你才有可能搭建自己的系统。

4. 克服“完美主义”与“恐惧”，建立小步快跑的反馈闭环

在左象限（尤其是优秀技术人聚集的 S 象限）呆久了的人，很容易陷入“完美主义墙”——总觉得自己的产品、技术或投资研究还得再完美一点才敢开始，或者因为害怕亏损而迟迟不敢迈出第一步。

  • 富人的做法： 接受不完美，管理风险而不是逃避风险。
  • 日常训练：
    • 在投资上： 拿出一笔“即使亏光也完全不影响生活”的极小资金去实操。只有你真金白银买入一个资产时，你对市场的敏锐度、对财报的关注度才会真正被激活。
    • 在商业上： 贯彻 MVP（最小可行性产品） 逻辑。有一个想法，用最快、成本最低的方式去市场上测试（比如写一篇博客看有没有人看，做个简单的 Demo 看有没有人愿意用），根据市场的真实反馈来修正，而不是闭门造车。

你可以从今天开始做一个小练习：先支付自己。 每次收到收入（比如薪水）时，第一件事不是去还信用卡或计划消费，而是雷打不动地拿出一笔固定的比例（哪怕是 10%）直接存入你的“资产建设账户”（用于投资或学习投资）。用剩下的钱去应付生活，如果不够，逼自己去想办法赚外快，而不是动用那笔资产基金。

[!NOTE]

显存（VRAM）是决定模型能否跑起来的绝对硬限制（OOM 闸门），而算力（TFLOPS）和显存带宽则决定了它能跑多快。

模型是否跑起来——显存计算

1.推理场景(inference)

推理时的显存占用主要由三部分组成：静态权重、KV Cache（动态上下文）*和*框架固有开销。 \[ \text{Total VRAM (Inference)} = M_{\text{weight}} + M_{\text{kv\_cache}} + M_{\text{overhead}} \]

  • 静态权重 (\(M_{\text{weight}}\))： \[ M_{\text{weight}} = P \times b_{\text{precision}} \times 1.2 \]

    • \(P\) 为参数量（单位：Billion，十亿）。
    • \(b_{\text{precision}}\) 为每个参数占用的字节数（BF16/FP16 = 2 Byte，INT8 = 1 Byte，INT4 = 0.5 Byte）。
    • 1.2 是预留的 20% 缓冲区（用于存放激活值中间状态和对齐开销）。

  • KV Cache (\(M_{\text{kv\_cache}}\))： 大模型在多轮对话中缓存历史 Key 和 Value 矩阵的开销。 \[ M_{\text{kv\_cache}} = 2 \times B \times S \times L \times H \times b_{\text{precision}} \]

    • \(B\): Batch Size（批次大小）

    • \(S\): Sequence Length（总上下文长度，输入+输出）

    • \(L\): 层数 (Layers)

    • \(H\): 隐藏层维度 (Hidden Size)

      (注意：如果模型采用了 GQA

2.训练/微调

训练是显存杀手。除了静态权重，你必须把梯度、优化器状态和激活值全部存下来（为了反向传播）。

以主流的 AdamW 优化器 + 混合精度 (FP16/BF16) 训练 为例，每 1B 参数所需的模型状态（Model States）显存高达 16 GB：

  • Model Weights (FP16): 2 bytes/param
  • Gradients (FP16): 2 bytes/param
  • Optimizer States (AdamW): 12 bytes/param (包含 FP32 Master Weights 4 bytes + 一阶动量 4 bytes + 二阶动量 4 bytes)

\[ \text{Model States VRAM} = P \times 16 \text{ GB} \]

に加え、激活值 (Activations) 会随着 Batch Size 和 Sequence Length 的增大呈线性爆发，通常需要开启梯度检查点 (Gradient Checkpointing) 来空间换时间，减少 70% 的激活值显存。

硬件选型

在实际工程落地中，我们会根据模型规模和业务场景，将显卡设备划分为以下几个梯队：

显卡型号 显存 (VRAM) 显存带宽 核心定位与适用场景

1.快速自动化评估工具

Hugging Face Accelerate CLI: 可以在不真正下载完整模型权重（只读 config.json）的情况下，通过元设备模拟加载并计算出所需的精准显存。

它会为你输出该模型在 float16 下的单层最大显存、整机加载显存以及使用 Adam 训练所需的显存量。

2.工程checklists
  • CUDA Context Base Overhead（底噪显存） PyTorch 或 JAX 在初始化 CUDA 时，哪怕一个模型都不加载，也会瞬间吃掉大概 1.2 GB - 2 GB 的显存作为 Context 底噪。因此，计算出的理论剩余显存如果小于 2GB，必定 OOM。
  • FlashAttention 的必要性 在 Transformer 结构的推理和训练中，原始 Attention 的显存复杂度是 \(O(S^2)\)。工程部署上必须强制开启 FlashAttention-2 或 PyTorch 的 SDPA，将显存复杂度降到 \(O(S)\)，否则长文本下 KV Cache 会瞬间撑爆显存。
  • 量化（Quantization）的边界 如果显存卡在临界点（例如 24GB 显卡想跑 14B 模型），首选 AWQ 或 GPTQ 进行 4-bit / 8-bit 推理量化。INT4 量化可以将显存直接斩断至原来的 25%，而综合泛化精度损失通常在 1% 左右，完全在工业可接受范围内。
3.多卡分布式切分策略（当单卡装不下时）
  • 流水线并行 (Pipeline Parallelism - PP): 按层切分，适合单卡显存能装下若干层，但装不下整个模型。
  • 张量并行 (Tensor Parallelism - TP): 将单层的矩阵乘法切分到多卡（如 Megatron-LM），对卡间通信带宽（NVLink）要求极高。如果在没有 NVLink 的 PCIe 机器上跑 TP，通信延迟会彻底拖垮 TFLOPS。
  • ZeRO (Zero Redundancy Optimizer): 如果是训练阶段，强烈推荐 DeepSpeed 的 ZeRO-3 或者是 JAX 的 Sharding 机制，将优化器状态、梯度和权重分摊到所有 GPU 节点。
